# 当前任务

## TASK-076：产品中心广告调优对象 API 状态固化

状态：已完成。

背景：
- 销售表现接口同步的是销售产品池，不等于 SP 广告投放产品池。
- TASK-071 到 TASK-075 已在前端把产品中心拆成“广告调优对象 / 销售档案 / 全部产品”。
- 当前对象边界主要由前端根据归因数量和 SP 指标临时计算，后续页面或脚本仍可能误把销售档案产品当作广告调优对象。

目标：
- `/api/products` 和 `/api/products/{product_id}` 直接返回产品广告覆盖状态。
- 状态只做派生判断，不新增表、不写真实库：
  - `attributed`：有活跃人工归因规则。
  - `sp_unattributed`：无活跃归因规则，但当前周期有 SP 指标。
  - `not_advertised`：当前系统暂无 SP 覆盖证据，只作为销售档案。
- API 同时返回 `is_ad_tuning_eligible`，只有 `attributed` 和 `sp_unattributed` 为真。
- 前端优先使用 API 字段，保留现有本地计算作为兼容兜底。
- 不新增任何广告自动执行能力。

允许修改：
- `backend/app/api/routes/products.py`
- `frontend/src/api.ts`
- `frontend/src/App.tsx`
- `scripts/smoke_product_ad_object_scope_api.py`
- `当前任务.md`
- `project_state.json`
- `开发任务清单.md`
- `项目进展.md`

禁止修改：
- `.env`
- `.env.example`
- `frontend/.env.local`
- `data/app.db`
- 任何广告执行接口或自动投放能力

成功标准：
- 产品 API 返回 `ad_coverage_status` 和 `is_ad_tuning_eligible`。
- 有活跃归因规则的产品返回 `attributed` 且可进入广告调优。
- 无归因但有 SP 指标的产品返回 `sp_unattributed` 且可进入广告调优。
- 无归因且无 SP 指标的销售产品返回 `not_advertised` 且不可进入广告调优。
- 前端 `getProductAdCoverageStatus` 优先使用 API 的 `ad_coverage_status`。
- 前端 `isProductAdTuningEligible` 优先使用 API 的 `is_ad_tuning_eligible`。
- 前端仍保留本地计算兜底，避免旧接口响应导致页面崩溃。
- 源码不出现任何自动执行广告动作字段或按钮。
