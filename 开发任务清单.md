# 开发任务清单

## 执行原则

- 每次只执行一个任务。
- 任务必须有明确目标、允许文件、禁止文件、验收命令和成功标准。
- 验收通过前不能进入下一任务。
- 本系统只做决策辅助，不做广告自动执行。

## TASK-002：审查并收敛 SP 异常规则引擎

状态：已完成。

目标：
- 基于近 14 天 SP 关键词 / 搜索词数据生成异常事件。
- 规则结果必须可溯源。
- 输出只允许是异常和建议线索，不允许出现广告自动执行动作。

前置条件：
- TASK-001 已完成。
- SQLite 中已有 SP 指标数据。

允许修改：
- `backend/app/services/rule_service.py`
- `backend/app/models/anomaly.py`
- `backend/app/api/routes/anomalies.py`
- `scripts/smoke_rule_engine.py`
- `tests/test_rule_service.py`

禁止修改：
- `.env`
- `.env.example`
- `backend/app/services/gerpgo_client.py`
- `backend/app/services/sync_service.py`
- `frontend/`

输入：
- `sp_keyword_metrics`
- `sp_search_term_metrics`
- 默认周期：近 14 天
- 产品目标：人工设置

输出：
- `anomaly_events`
- `rule_result_json`
- `evidence_json`

第一版异常类型：
- ACOS 异常
- 点击多无订单
- CVR 下滑
- 花费异常
- 曝光异常
- 库存目标冲突

验收命令：

```powershell
python scripts/smoke_rule_engine.py
```

成功标准：
- 能基于 SQLite 生成异常事件。
- 每个异常事件包含周期、对象、指标、规则命中原因。
- 规则输出不包含自动执行动作字段。
- smoke 脚本输出成功状态和异常数量。

失败停止条件：
- 缺少真实字段导致规则无法判断。
- 需要改动积加 API 同步逻辑。
- 需要改动前端。
- 验收命令失败。
